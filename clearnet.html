How does the test work?
A connection to the clearnet is attempted using two different methods (iframe and loading JavaScript).
Should the iframe test work and be able to connect to the clearnet, a message will appear above this text.
If the JavaScript test works and is able to connect to the clearnet, a zeronet notification will be displayed.
Both messages contain your presumed IP address. I do not actively save the IP address
(it is always possible that an ISP or hoster records the traffic).
How can i prevent it?
A solution suggestion would be that you use a separate browser to visit zites,
which does not allow connection outside 127.0.0.1 or localhost.
For example, you could enable a proxy in Firefox and specify the address of
a non-existent proxy (e.g. host: 127.0.0.1, port: 43321) and
only add exceptions for 127.0.0.1 or localhost.
